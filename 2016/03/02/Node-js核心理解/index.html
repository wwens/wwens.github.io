Node.js核心理解

March 02 2016

本文整理于朴灵老师在InfoQ上的演讲 Node的设计与实现

“Node.js is a platform built on Chrome’s JavaScript runtime for easily building fast, scalable network applications. Node.js uses an event-driven, non-blocking I/O model that makes it lightweight and efficient, perfect for data-intensive real-time applications that run across distributed devices.”

Node.js的三个关键词：v8，事件驱动，非阻塞io模型。

问题2：什么是事件驱动（event loop）？

Event Loop主要做两件事情：检查事件、处理事件。

检查事件 － Watcher
向Watcher询问是否有事件需要处理
处理事件 － Handles
由Watcher产生具体要处理的事件
handle执行传入的回调函数
在Event Loop中没有Watcher的时候退出进程
Event Loop —> Watcher —>Handles
单线程阻塞问题
Non-Blocking I/O

问题3：单线程阻塞的解决方案？如何执行事件？

磁盘I/O和网络I/O的访问耗时远远大于内存和CPU缓存的访问耗时
磁盘I/O和网络I/O的访问阻塞CPU的执行
进行磁盘I/O和网络I/O时，CPU浪费
后续计算无法进行

解决方案：
非阻塞执行：CPU与I/O并行执行，计算设备与I/O设备互补干扰
通过线程池结合事件驱动实现
没有完美的非阻塞I/O

问题4：V8的意义？

V8引擎大大提高了Javascript的执行效率，可以说，没有V8就没有Node。

问题5：单线程带来的多核CPU利用问题？

  • 进程间消息传递
  • 进程间不共享数据
  • 通过消息传递
  • 父进程 <—— message ——> IPC <—— message ——> 子进程
  • 子进程／Cluster：分离监控和业务
