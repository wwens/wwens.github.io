web性能测试

March 16 2016

性能测试(Performance Testing)

  1. 负载测试：Load Testing — 用户数量，性能指标
  2. 压力测试：Stress Testing — 高压力，系统是怎么死的，预案
    (内存泄漏，网络堵塞，CPU100%，线程资源耗尽，数据库异常)
  3. 容量测试: Volume Testing — 最大支持数量

性能指标：

  1. 响应时间：Response Time — 从用户角度评价系统的处理速度，经验值（2秒，5秒，10秒）
  2. 吞吐量／吞吐率：
    IO => 硬盘IO、网络IO、CPU内存、请求处理能力，打开页面数量
  3. 事务处理能力：Tps（Transaction Per Second)
    事务 => 打开一个也页面，实现登陆，实现发帖、回帖，实现一笔网购，我和别人聊天，支付，充电话费

关注点（三层架构 客户端－服务器－数据库，催生集群、云计算、虚拟化）：

  1. 响应时间，服务器端的处理速度
  2. 服务器端的资源使用情况（硬件、带宽…）
  3. 数据库端的资源使用情况
  4. 最大访问用户数
  5. 最大核心业务处理数
  6. 系统能否支持7*24小时运转
  7. 内存资源，线程资源能否正常回收
  8. 代码：算法，SQL
  9. 稳定性，可恢复性

核心原理（网络分布式架构）

  1. 基于协议（不基于界面、代码）
  2. 多线程
  3. 模拟真实场景
